# Flower Production Features Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform Verdant from a general garden planner into a complete commercial flower production tool covering planning, growing, ordering, sales tracking, and season-over-season optimization.

**Architecture:** Extend the existing Quarkus+Kotlin backend with new entities, services, and endpoints. The React web app gets new pages and a calendar view. The Android app gets updated models and screens. Since no production data exists, the database schema is rewritten from scratch — all 9 existing migrations are consolidated into a single V1 and new tables are added.

**Tech Stack:** Quarkus + Kotlin (backend), React + TypeScript + Tailwind (web), Kotlin + Jetpack Compose (Android), PostgreSQL, Flyway migrations.

**Important notes:**
- All new tables use `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` to match existing tables (NOT `BIGSERIAL`).
- All `unit_type` fields use the `UnitType` enum consistently (not raw Strings).
- Weather/GDD: For now, frost dates and growing degree base temp are stored on the Season entity. Full weather data collection (daily temps, GDD accumulation) is deferred to a future phase. The season frost dates are sufficient for sow-date calculations.
- Flyway: Since all existing migrations are replaced, run `flyway clean` on dev databases before applying. In dev profile, set `quarkus.flyway.clean-at-start=true`.
- Bed history authorization: `GET /api/beds/{id}/history` must authorize through the garden ownership chain (bed → garden → garden.ownerId) since `bed` has no `user_id` column.
- Production forecasting with missing species data: When `expectedStemsPerPlant`, `germinationRate`, or `daysToHarvest` are null, use sensible defaults (1 stem/plant, 80% germination, 90 days) and include a warning in the response explaining which defaults were assumed.

```sql
-- Season: groups all activity for a year/period
CREATE TABLE season (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES app_user(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,          -- e.g. "2026", "Spring 2026"
    year INTEGER NOT NULL,               -- 2026
    start_date DATE,                     -- optional explicit start
    end_date DATE,                       -- optional explicit end
    last_frost_date DATE,                -- spring frost-free date
    first_frost_date DATE,               -- fall first frost
    growing_degree_base_c DOUBLE PRECISION DEFAULT 10.0,
    notes TEXT,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now(),
    UNIQUE(user_id, name)
);

-- Customer: where flowers go (florists, markets, CSA, weddings)
CREATE TABLE customer (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES app_user(id) ON DELETE CASCADE,
    name VARCHAR(200) NOT NULL,
    channel VARCHAR(50) NOT NULL,        -- FLORIST, FARMERS_MARKET, CSA, WEDDING, WHOLESALE, DIRECT, OTHER
    contact_info TEXT,
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT now(),
    UNIQUE(user_id, name)
);

-- Pest/disease log: structured observations
CREATE TABLE pest_disease_log (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES app_user(id) ON DELETE CASCADE,
    season_id BIGINT REFERENCES season(id) ON DELETE SET NULL,
    bed_id BIGINT REFERENCES bed(id) ON DELETE SET NULL,
    species_id BIGINT REFERENCES species(id) ON DELETE SET NULL,
    observed_date DATE NOT NULL DEFAULT CURRENT_DATE,
    category VARCHAR(20) NOT NULL,       -- PEST, DISEASE, DEFICIENCY, OTHER
    name VARCHAR(200) NOT NULL,          -- e.g. "Aphids", "Powdery mildew", "Iron deficiency"
    severity VARCHAR(10) NOT NULL DEFAULT 'MODERATE', -- LOW, MODERATE, HIGH, CRITICAL
    treatment TEXT,
    outcome VARCHAR(20),                 -- RESOLVED, ONGOING, CROP_LOSS, MONITORING
    notes TEXT,
    image_url TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Variety trial: side-by-side comparison across seasons
CREATE TABLE variety_trial (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES app_user(id) ON DELETE CASCADE,
    season_id BIGINT NOT NULL REFERENCES season(id) ON DELETE CASCADE,
    species_id BIGINT NOT NULL REFERENCES species(id) ON DELETE CASCADE,
    bed_id BIGINT REFERENCES bed(id) ON DELETE SET NULL,
    plant_count INTEGER,
    stem_yield INTEGER,                  -- total stems harvested
    avg_stem_length_cm INTEGER,
    avg_vase_life_days INTEGER,
    quality_score INTEGER,               -- 1-10 subjective rating
    customer_reception VARCHAR(20),      -- LOVED, LIKED, NEUTRAL, DISLIKED
    verdict VARCHAR(20) NOT NULL DEFAULT 'UNDECIDED', -- KEEP, EXPAND, REDUCE, DROP, UNDECIDED
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Bouquet recipe: saved arrangements
CREATE TABLE bouquet_recipe (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES app_user(id) ON DELETE CASCADE,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    image_url TEXT,
    price_cents INTEGER,
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE bouquet_recipe_item (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    recipe_id BIGINT NOT NULL REFERENCES bouquet_recipe(id) ON DELETE CASCADE,
    species_id BIGINT NOT NULL REFERENCES species(id) ON DELETE CASCADE,
    stem_count INTEGER NOT NULL,
    role VARCHAR(30) DEFAULT 'FLOWER',   -- FLOWER, FOLIAGE, FILLER, ACCENT
    notes TEXT
);

-- Succession schedule: recurring sow pattern
CREATE TABLE succession_schedule (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES app_user(id) ON DELETE CASCADE,
    season_id BIGINT NOT NULL REFERENCES season(id) ON DELETE CASCADE,
    species_id BIGINT NOT NULL REFERENCES species(id) ON DELETE CASCADE,
    bed_id BIGINT REFERENCES bed(id) ON DELETE SET NULL,
    first_sow_date DATE NOT NULL,
    interval_days INTEGER NOT NULL,      -- e.g. 14 for biweekly
    total_successions INTEGER NOT NULL,  -- e.g. 6 rounds
    seeds_per_succession INTEGER NOT NULL,
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Production target: "I need X stems/week of Y during Z period"
CREATE TABLE production_target (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES app_user(id) ON DELETE CASCADE,
    season_id BIGINT NOT NULL REFERENCES season(id) ON DELETE CASCADE,
    species_id BIGINT NOT NULL REFERENCES species(id) ON DELETE CASCADE,
    stems_per_week INTEGER NOT NULL,
    start_date DATE NOT NULL,            -- delivery window start
    end_date DATE NOT NULL,              -- delivery window end
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
);
```
